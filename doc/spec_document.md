# spec_document.md — 7열 드릴러(가제) 싱글 무한모드 프로토타입 스펙 (v0.2)

작성일: 2026-03-05 (Asia/Seoul)  
목표: **Codex가 바로 구현 가능한** “싱글 무한모드” 프로토타입 스펙.  
범위: 싱글플레이 코어(그리드/이동=채굴/점프/중력/블록 낙하/피격/HP). 멀티플레이는 제외.

---

## 0) 핵심 요약

- 맵: **가로 7칸 고정**, 아래로 무한 생성.
- 플레이어: **1블록(1타일) 크기**, 기본 HP **3**.
- 입력: 상/하/좌/우.
- 공격 버튼 없음. **좌/우/하 방향으로 블록 쪽으로 “이동 시도” = 채굴**.
- **상(↑)** 입력:
  - 위가 **빈칸**이면: **1칸 점프**(공중 상태) → 이후 중력으로 떨어짐. 공중에서 좌/우 이동 가능. 착지 전 **재점프 불가**.
  - 위가 **블록**이면: **상향 채굴 가능**(규칙은 아래 “상향 채굴” 참조).
- 블록 종류 4개: 기본(HP1), 튼튼(HP2), 무적(파괴 불가), 이벤트(placeholder).
- **떠 있는 블록(아래가 빈칸)**은 **0.1초 흔들림 후 낙하**. 낙하는 **가속도 0에서 시작하여 점점 빨라짐**.
- 낙하 블록이 플레이어를 맞추면: 플레이어는 **옆으로(빈칸이 있는 쪽)** 밀려나며 **HP -1**.

## 1) 용어 및 좌표

- 타일 좌표: `(x, y)`
  - `x ∈ {0..6}` (가로 7칸)
  - `y ∈ ℤ` (아래로 증가). 화면 상단이 더 작은 y, 아래가 더 큰 y.
- “빈칸” = 해당 타일에 블록이 없는 상태.
- “블록” = BASIC/STURDY/UNBREAKABLE/EVENT 타일 오브젝트.
- 플레이어는 **정확히 1타일**을 점유한다. (충돌 판정도 타일 단위 + 낙하 중에는 연속 좌표 보조)

---

## 2) 화면/아트(플레이스홀더)

- 플레이어: **삼각형(▲)**  
  - **바라보는 방향/진행 방향**을 향해 회전한다.
  - 기본은 마지막으로 유효한 이동 입력 방향(좌/우/하/상)을 **Facing**으로 저장.
- 블록: 사각형 컬러 타일
  - BASIC: 단색(예: 회색)
  - STURDY: 단색 + HP1 상태일 때 “크랙(금)” 표시
  - UNBREAKABLE: 진한 단색(예: 검정)
  - EVENT: 별도 색(예: 노랑) + 아이콘 placeholder 가능

> 아트/사운드는 MVP에서 최소화. 중요한 건 피드백(흔들림/피격) 타이밍이 보이게 하는 것.

---

## 3) 게임 루프(권장)

- 고정 업데이트 권장: `dt = 1/60` (60 FPS)
- 처리 순서(권장):
  1. 입력 수집(방향키/스와이프)
  2. 플레이어 액션 처리(이동/채굴/점프)
  3. 블록 “불안정(아래 빈칸)” 체크 → 흔들림 타이머 갱신 → 낙하 상태 전환
  4. 낙하 블록 물리 업데이트(속도/위치)
  5. 낙하 블록 ↔ 플레이어 충돌 처리(피격, 밀침)
  6. 낙하 블록 착지/정지 처리(그리드 스냅)
  7. 깊이(Depth)/점수 갱신
  8. 카메라 추적/렌더

## 4) 플레이어 상태/스탯

### 4.1 체력(HP)
- 기본 HP: **3**
- 최대 HP: 3 (MVP 고정; 확장 가능)
- HP 감소 원인(MVP):
  - **낙하 중인 블록에 맞음**: HP -1

### 4.2 상태(State)
- `Grounded` (지면/블록 위에 서 있음)
- `Airborne` (점프 이후 공중, 또는 상향 채굴로 올라갔다가 아래가 빈칸인 상태 포함)
- 공중 규칙:
  - 공중에서는 **좌/우 이동 가능**
  - 공중에서는 **상(점프) 불가** (재점프 금지)
  - 하(↓) 입력은 “공중에서 1칸 하강”을 허용할지 여부는 **Open Question**(기본값: 공중에서는 하 입력 무시)

## 5) 입력 → 행동 규칙

입력은 `dir ∈ {UP, DOWN, LEFT, RIGHT}`.

### 5.1 공통: 맵 경계
- 좌/우 이동은 `x=0..6` 범위 내에서만 가능.
- 범위를 벗어나려 하면 이동 실패(튕김 피드백만).

---

## 6) 타일/블록 타입

블록은 다음 필드를 가진다.

```ts
type BlockType = "BASIC" | "STURDY" | "UNBREAKABLE" | "EVENT";

interface Block {
  type: BlockType;
  hp: number | null;     // BASIC=1, STURDY=2, UNBREAKABLE=null, EVENT=1(또는 null)
  eventId?: string;      // EVENT만 (placeholder)
  // 낙하 관련 상태
  fallState: "STATIC" | "SHAKING" | "FALLING";
  shakeTimer: number;    // seconds, SHAKING일 때 0.1까지
  vy: number;            // 타일/초, FALLING일 때
  yFloat: number;        // 연속 좌표(타일 기준), FALLING일 때만 사용. 예: y=10.0 시작
}
```

## 7) 이동 = 채굴 (좌/우/하)

플레이어가 **좌/우/하**로 “이동 시도”할 때, 목표 타일이 블록이면 채굴 로직으로 처리한다.

### 7.1 목표 타일이 빈칸
- 즉시 이동 성공:
  - `player.x/y` 갱신
  - Facing = 입력 방향
  - 만약 이동 후 아래가 빈칸이면(지지 없음) → `Airborne`로 전환(자연 낙하)

### 7.2 목표 타일이 BASIC (HP1)
- 1회 시도 시:
  - 블록 파괴(빈칸으로 변경)
  - **그 칸으로 이동 성공**
  - Facing 갱신

### 7.3 목표 타일이 STURDY (HP2)
- 1회 시도 시:
  - 블록 HP -1
  - 플레이어는 **이동하지 않음**(제자리)
  - 튕김/타격 피드백(짧은 넉백 연출은 선택)
- 2회 시도(HP=1 상태에서) 시:
  - 블록 파괴
  - **그 칸으로 이동 성공**

### 7.4 목표 타일이 UNBREAKABLE
- 파괴 불가
- 플레이어 이동 실패(튕김 피드백)

### 7.5 목표 타일이 EVENT
- placeholder:
  - 기본 동작: **EVENT는 HP1처럼 1회 파괴 + 이동 성공**
  - 동시에 `eventId`를 기반으로 이벤트 콜백 실행(지금은 로그 출력만)

> EVENT의 “파괴/유지/변환”은 향후 확장. MVP는 단순 파괴+이동으로 통일.

## 8) 상(↑) 입력: 점프 vs 상향 채굴

상 입력은 **상단 타일 상태**에 따라 갈린다.

### 8.1 위가 빈칸이면: 1칸 점프
- 조건:
  - `tile(x, y-1)`이 빈칸
  - 플레이어가 `Grounded`일 때만 가능 (재점프 금지)
- 처리:
  - 플레이어는 즉시 `y = y - 1`로 이동(점프 높이 1칸)
  - 상태를 `Airborne`로 전환
  - Facing = UP
  - 이후 중력으로 아래로 내려온다(“9) 플레이어 중력/착지” 참조)

### 8.2 위가 블록이면: 상향 채굴(UP Mining)
- 조건:
  - `tile(x, y-1)`이 블록
  - 플레이어가 `Grounded`일 때만 처리(권장). (공중에서 위 채굴 허용 여부는 Open Question)
- 처리(권장 규칙: 좌/우/하와 동일한 채굴 규칙을 “위”에도 적용):
  - 위 블록 타입별로 HP 처리
  - 블록이 파괴되면:
    - **즉시 1칸 위로 이동(상승)**하여 그 칸을 점유한다.
    - 상태는 아래 지지 여부에 따라 결정:
      - `tile(x, y+1)`(원래 자리)이 빈칸이므로 대부분 `Airborne`
  - 블록이 파괴되지 않으면:
    - 이동 없음(튕김/타격 피드백)

> 참고: “위로는 블럭이 없다면 파고 갈 수 없음”이라는 기존 조건은,  
> 점프 도입 이후에는 “점프는 빈칸 위로만 가능 / 블록 위로는 채굴로만 가능”으로 재정의한다.

## 9) 플레이어 중력/착지

### 9.1 중력(플레이어)
- 플레이어는 `Airborne` 상태이면 매 프레임 아래 조건을 평가:
  - 아래 타일 `tile(x, y+1)`이 **빈칸**이면:
    - 일정 시간/속도로 하강(연속 좌표로 구현해도 되고, MVP는 “프레임당 1칸 하강” 대신 **타이머 기반 하강** 권장)
  - 아래 타일이 **블록(또는 바닥)**이면:
    - 착지 → `Grounded`

> 구현 난이도 절충안(MVP): 플레이어는 공중일 때 `fallCooldown`(예: 0.08s)을 두고, 쿨다운마다 1칸씩 내려오게 만들어도 된다.  
> (낙하 블록은 “연속 가속”이 요구되므로 블록 낙하는 별도로 연속 좌표로 구현)

## 10) 블록 낙하 시스템

### 10.1 불안정 조건
어떤 블록이든(기본/튼튼/무적/이벤트) 아래 타일이 빈칸이면 “불안정”으로 본다.

- 조건: 블록이 `STATIC`이고 `tile(x, y+1)`이 빈칸이면
  - `fallState = SHAKING`
  - `shakeTimer = 0`

### 10.2 흔들림(Shake) → 낙하(Fall)
- SHAKING 상태에서:
  - `shakeTimer += dt`
  - `shakeTimer >= 0.1` 이면:
    - `fallState = FALLING`
    - `vy = 0`
    - `yFloat = y` (현재 타일 y를 실수로 보관)
    - 해당 타일의 “그리드 점유”는 **FALLING 오브젝트로 이동** (그리드에서는 빈칸 취급)

### 10.3 낙하 물리(연속 가속)
- FALLING 상태에서 매 프레임:
  - `vy += g * dt` (g는 양수, 아래 방향)
  - `yFloat += vy * dt`
- 권장 상수:
  - `g = 60 tiles/s^2` (조정 가능)
  - `vy` 상한(터널링 방지): 예 `vyMax = 25 tiles/s`

### 10.4 착지/정지
낙하 블록의 “아래”에 착지할 바닥이 생기면 정지한다.

- 현재 낙하 블록의 좌표:
  - `yCell = floor(yFloat)` 또는 “블록의 하단” 기준으로 계산(간단화를 위해 블록은 1x1 정사각형)
- 착지 조건(권장):
  - 블록이 내려가다가, 다음 정수 타일 경계에서
    - `tile(x, yBelow)`가 **블록**이거나,
    - `yBelow`가 월드 바닥(없다면 무한)인 경우
  - 그러면:
    - 낙하 블록의 위치를 가장 가까운 그리드 `y = yBelow - 1`로 스냅
    - 그리드에 다시 “STATIC 블록”으로 배치
    - fallState=STATIC, shakeTimer=0, vy=0

## 11) 낙하 블록 ↔ 플레이어 충돌/피격

### 11.1 충돌 판정(권장 단순화)
- 낙하 블록은 연속 `yFloat`를 가지지만, MVP에서는 아래처럼 단순 판정을 권장:
  - 낙하 블록의 “현재 점유 셀”을 `fallCellY = round(yFloat)` 또는 `floor(yFloat)`로 계산
  - 만약 `(block.x == player.x && fallCellY == player.y)` 가 성립하면 “피격” 처리

(더 정확히 하려면 AABB 교차로 처리 가능)

### 11.2 피격 처리
- 피격이 발생하면:
  1. **플레이어 HP -1**
  2. 플레이어를 **좌/우 중 “빈칸이 있는 쪽”으로 1칸 밀어냄**
     - 우선순위 규칙(권장, 결정적):
       - 왼쪽이 빈칸이고 오른쪽이 블록 → 왼쪽으로
       - 오른쪽이 빈칸이고 왼쪽이 블록 → 오른쪽으로
       - 양쪽 다 빈칸 → **플레이어 Facing 방향의 반대쪽**으로(기본값) 또는 **오른쪽**(고정)
       - 양쪽 다 블록/벽(이동 불가) → **Open Question** (기본값: 이동 실패, HP만 감소)
  3. 피격 연출:
     - 0.2초 무적(연속 피격 방지) `iFrames = 0.2s` 권장 (MVP 안정성)
  4. HP <= 0이면 게임오버

### 11.3 낙하 블록의 처리
피격 후 낙하 블록이 어떻게 되는지(플레이어를 밀치고 계속 떨어질지/그 자리에 멈출지)는 명확히 정의가 필요함.

- MVP 권장 규칙(단순/자연):
  - **블록은 계속 낙하**한다.
  - 피격 순간에 플레이어가 옆으로 밀려났다면, 블록은 해당 셀을 통과하여 아래로 계속 내려감.
  - (단, 속도가 너무 빨라져 터널링 위험이 있으면 vyMax 적용)

## 12) 무한 맵 생성(싱글)

- 세로를 청크로 관리(예: 24줄 단위).
- `seed` 기반 결정적 생성 권장:
  - `seed`는 현재는 랜덤 32-bit 정수.
  - `chunkIndex`로 PRNG를 재시드하여 해당 구간 타일을 생성.
- 생성 규칙(초안, 밸런싱 대상):
  - BASIC: 70%
  - STURDY: 20%
  - UNBREAKABLE: 8%
  - EVENT: 2% (placeholder)
- 소프트락 방지(권장 최소):
  - 각 새 줄 생성 시 **최소 1칸은 “빈칸”**을 보장하거나,
  - 최소 1개의 “하강 가능 경로”가 생기도록 열 단위로 편향 생성.

> 낙하/채굴 때문에 지형이 실시간으로 변하므로, “완전한 경로 보장”은 어렵다. MVP는 최소한 “완전 막힘”이 나오지 않게 확률/규칙을 보수적으로 시작.

   - 현재 가정: 이동은 못 하고 HP만 감소(그리고 i-frames)  
   - 혹은 즉사/추가 피해/블록이 멈춤 등의 규칙이 필요?
4. **무적블록 낙하 여부**: 스펙에서는 “모든 블록은 아래가 비면 낙하”로 가정했음. 무적블록도 떨어지게 할까?
5. **플레이어의 자연 낙하 속도**: 블록처럼 연속 가속으로 할지, 타일 단위 타이머 낙하로 단순화할지 선택 필요. (현재는 타이머 방식도 허용)

## 16) 완료 정의(Prototype Acceptance Criteria)

- 7열 무한 맵에서 플레이어가 이동/채굴/점프 가능
- BASIC/STURDY/UNBREAKABLE/EVENT가 규칙대로 동작
- 블록이 아래 빈칸이면 0.1초 흔들리고 가속 낙하
- 낙하 블록이 플레이어를 맞추면 HP-1 및 옆으로 밀침
- HP 0이면 게임오버 및 재시작 가능
- Depth/BestDepth 표시
